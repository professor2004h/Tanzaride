🚀 DEPLOYMENT STEPS FOR HOSTINGER
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

STEP 1: Access Your Hosting Control Panel
   → Log in to your Hostinger account
   → Go to your hosting control panel (hPanel)
   → Select your domain (tanzaride.com)

STEP 2: Access File Manager
   → Click on "File Manager" in the control panel
   → Navigate to the public_html folder (or www folder)
   → This is where your website files will go

STEP 3: Upload Files
   Option A - Using File Manager:
   → Click "Upload Files" button
   → Select all HTML files (index.html, about.html, etc.)
   → Upload style.css
   → Upload main.js
   → Create "assets" folder
   → Upload TanzaRide-Logo.webp to the assets folder

   Option B - Using FTP:
   → Use FileZilla or any FTP client
   → Connect using your FTP credentials from Hostinger
   → Upload all files maintaining the folder structure

STEP 4: Verify File Structure
   Your public_html folder should look like this:
   public_html/
   ├── index.html
   ├── about.html
   ├── howitworks.html
   ├── cars.html
   ├── listcar.html
   ├── contact.html
   ├── privacy.html
   ├── terms.html
   ├── style.css
   ├── main.js
   └── assets/
       └── TanzaRide-Logo.webp

STEP 5: Test Your Website
   → Visit https://tanzaride.com in your browser
   → Test all navigation links
   → Test on mobile device
   → Verify forms are working (they show alerts)
   → Check all pages load correctly


🔧 TROUBLESHOOTING
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Problem: Website shows "Index of /" or directory listing
Solution: Make sure index.html is in the public_html folder

Problem: Logo not showing
Solution: Verify the assets folder is uploaded and contains TanzaRide-Logo.webp

Problem: CSS not loading (website looks unstyled)
Solution: Check that style.css is in the same folder as index.html

Problem: Navigation not working on mobile
Solution: Verify main.js is uploaded correctly

Problem: 404 errors on page links
Solution: Ensure all HTML files are in the root directory (public_html)


📧 FORM CONFIGURATION (OPTIONAL)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Currently, forms show JavaScript alerts. To make them send emails:

Option 1 - Use FormSpree (Easiest):
   1. Sign up at formspree.io
   2. Create a form endpoint
   3. Update form action in contact.html and listcar.html
   4. Example: <form action="https://formspree.io/f/YOUR_ID" method="POST">

Option 2 - Use Hostinger Email:
   1. Create a PHP file to handle form submissions
   2. Configure SMTP settings
   3. Update forms to submit to PHP file

Option 3 - Use Netlify Forms:
   1. Deploy to Netlify
   2. Add netlify attribute to forms
   3. Forms will automatically work
